## Why the formation-surface formula is exact

Across 766 train wells with non-degenerate ANCC:

```
within-well std of (TVT - (-Z + ANCC)):
  median 0.0065 ft
  p95    0.0072 ft
  max    0.0087 ft
```

That is *six thousandths of a foot*. The formula is a deterministic
coordinate transform, not an approximation. The same identity holds for all
six formation tops (`ASTNU`, `ASTNL`, `EGFDU`, `EGFDL`, `BUDA`) with
indistinguishable residual std — they are parallel surfaces along the
wellbore in the trajectory math. `b_well` varies ~2866 ft well-to-well so
it must be re-anchored per-well from the visible `TVT_input` prefix.

Once `ANCC(X, Y)` and `b_well` are known, `TVT` is fully determined.

## Where the error budget actually goes

Decomposition (measured in `bench/full_score.py smoke` on 200 wells):

```
Pure formula (no GBM):
  OOF RMSE = 27.93
  median well RMSE = 10.66
  max well RMSE = 252.88   <-- one outlier dominates

Formula features + 1 LGB seed (no beam, no XGB, no Ridge):
  OOF RMSE = 14.74
  median well RMSE = 9.71
  max well RMSE = 58.27    <-- GBM caught the outliers
```

The GBM doesn't make the formula more accurate; it learns *when not to
trust the formula* from the spatial uncertainty features (neighbor
distances, neighbor stds, plane-vs-row disagreement). Catastrophic wells
get pulled toward `last_known_TVT` (target=0).

## The v8 pipeline

`rogii/notebook/kaggle_cell_v8.py`. Single-cell Kaggle notebook with the
feature builder base64-embedded. Produces `/kaggle/working/submission.csv`.

1. **`FormationPlaneKNN`** (per-well centroids, K=10, IDW p=1, std-scaled
   X/Y, 2D weighted plane fit for all six tops).
2. **`RowKNN`** (~5M rows, K=20, IDW p=1, std-scaled X/Y, returns
   per-formation prediction + neighbor std + min distance).
3. **`build_hidden_features`** produces ~130 leakage-aware features per
   row in the hidden segment:
   - `fk_<F>` and `fk_<F>_dz` for all six tops (plane fit)
   - `knn_row_<F>`, `knn_row_<F>_dz`, `knn_row_<F>_std` for all six tops
   - Per-formation closed-form TVT delta `fk_tvt_formula_<F>` and
     `knn_row_tvt_pred_delta_<F>` (anchored at `last_known_TVT`)
   - `fk_b_<F>`, `fk_b_huber_<F>`, `knn_row_b_<F>`, `knn_row_b_huber_<F>`
     (median + Huber anchors per formation)
   - Inverse-variance multi-formation ensemble delta
     `knn_row_tvt_ensemble_delta`
   - GR rolling (5/21), grad, std, lag/lead, FFT
   - Beam-search Viterbi GR-vs-typewell alignment (cons + loose)
   - 11 typewell GR offset diffs at TVT shifts {-80,-40,-20,-10,-5,0,5,10,20,40,80}
   - prefix typewell-RMSE/MAE quality
   - prefix TVT step20/step100, MD/Z slopes
   - position deltas dx/dy/dz/dmd, slopes, dist_xy, dist_xyz
4. **GBMs**: LightGBM × 3 seeds (42, 7, 123) and XGBoost (seed 42), all
   GroupKFold(5, shuffle=True, seed=42), early_stopping=125. konbu17
   hyperparameters.
5. **Ridge stack** (positive, no intercept, alpha=1) — pick simple-avg
   vs Ridge by OOF.
6. **EWM(span=4)** per well on the reconstructed absolute TVT — smooths
   high-frequency GBM noise; the super-baseline (LB 12.602) trick.

## Enhancements over konbu17

| # | What | Expected gain |
| --- | --- | ---: |
| 1 | EWM(span=4) post-smoothing per well | +0.05-0.15 RMSE |
| 2 | Per-formation Huber-anchored b_well features | +0.05-0.15 |
| 3 | Per-formation closed-form TVT deltas (all 6) | small but free |
| 4 | Inverse-variance multi-formation ensemble feature | small |

These are independent levers, each cheap to add. Their combined effect is
unlikely to be additive (regularization via Ridge stack will down-weight
correlated signals), but +0.1-0.3 RMSE total is a defensible expectation.

## What is **not** in v8 (next iterations)

These are the highest expected-value items still on the table:

1. **Anisotropic kriging on ANCC** — Eagle Ford strikes NE-SW. konbu17
   uses isotropic IDW with X/Y std scaling (a weak isotropy correction).
   Proper anisotropic kriging with directional variogram should be worth
   0.3-0.6 RMSE. Implementation: estimate variogram empirically per
   formation, fit anisotropic Gaussian/exponential kernel, replace the
   IDW weighting in `RowKNN.impute`. Non-trivial but well-bounded.
2. **Particle filter / Kalman geosteering** — state `(TVT, dTVT/dMD)`
   evolving along MD; horizontal GR vs typewell GR is the observation;
   prior is the v8 formation-stack prediction. This matches the
   physical framing of the problem directly. Likely highest gain on
   the catastrophic outlier wells.
3. **Cross-section PNG vision encoder** — every well has a hand-drawn
   geosteering PNG. Train a small ConvNeXt-tiny to predict per-well dip /
   structure prior. Only useful as an extra feature; coarse anchor.
4. **Sub-zone Geology classification** — extend `geology_classifier.py`
   to the 22 sub-zone labels. Use the predicted sub-zone as a feature.

Order of operation: ship v8 first, see actual LB, then layer #1 if v8
underperforms expectation, otherwise continue down the list.
